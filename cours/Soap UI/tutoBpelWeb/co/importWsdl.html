Configuration des PartnerLinks du Process BPEL

Dans le cadre de l'orchestration d'un processus métier, l'orchestrateur peut être considéré comme le client des services web qu'il orchestre. Le processus doit donc connaître le "Port Type" des interfaces des services web afin de les déclarer en tant que Partner Link pour pouvoir, par la suite, invoquer leurs opérations et leurs assigner des paramètres d'entrée et de sortie.

Toutes les informations nécessaires à la déclaration des Partner Links sont contenu dans le WSDL File de chaque service web. Il est donc nécessaire d'importer dans notre projet BPEL le WSDL File des services web orchestrés.

Import des WSDL Files des services web dans le dossier bpelContent

Faites un clic droit sur le dossier "bpelContent" de votre BPEL Project puis sélectionnez "New > Other...", tapez "wsdl" dans le champ de filtre. Sélectionnez "WSDL File" puis cliquez sur "Next >"

Modifier le nom du fichier WSDL pour identifier qu'il est relatif à votre service web (WSAddition.wsdl par exemple) puis cliquez sur "Next >".

Le wizard vous propose des options de création d'un squelette de WSDL. Nous allons complètement écraser ce squelette par la suite. Ignorez donc cette étape et cliquez directement sur "Finish".

Votre WSDL File a bien été créé mais ne contient pas encore la définition du service web addition. Pour récupérer sa définition, vous avez besoin de son adresse réseau. Si vous utilisez un serveur JBoss EAP, il vous donne l'adresse de vos services web lors de leur déploiement. Copiez donc l'adresse du service web addition.

Collez l'adresse en URL d'un navigateur web en ajoutant "?wsdl" à la fin de l'adresse. La description du service web s'affiche alors dans le navigateur.

Affichez le code source de la page (crl+u ou clic droit > Code source de la page). Copiez la totalité du code source.

Retournez dans eclipse et collez la définition de votre service web dans votre WSDL File à la place du squelette généré par eclipse.

Répétez le même protocole avec vos deux autres services web afin que votre dossier bpelContent contienne le WDSL File de chaque web service de votre processus métier à orchestrer.

Import des PartnerLink dans le BPEL Process File

Lorsque votre fichier BPEL Process est ouvert en mode "Design", vous pouvez constater que juste à droite du schéma autogénéré par eclipse se trouve un panneau dans laquelle apparaissent notamment les Partner Links et les variables liées au processus d'orchestration. Pour le moment, l'unique Partner Link dont nous disposons est le client, c'est à dire l'orchestrateur lui même. De même, les seules variables disponibles sont "input" et "output" qui sont les variables d'entrée et de sortie de l'orchestrateur. Nous souhaitons ajouter trois nouveaux Partner Links nos trois services web. Nous allons commencer par le service addition. Cliquez sur le "+" vert à côté de "Partner Links". Un nouveau Partner Link apparaît. Donnez lui un nom évocateur comme "PartnerLinkAddition" par exemple.

Pour configurer un Partner Link nouvellement créé, cliquez dessus puis rendez-vous dans l'onglet "Details du panneau "Properties", puis cliquez sur "Browse...".

Un wizard de configuration de Partner Link s'ouvre. Cliquez directement sur "Add WSDL"

Un nouveau wizard s'ouvre par dessus le premier. Explorez votre projet pour trouver le WSDL Filede votre service web addition dans le dossier bpelContent. Sélectionnez le puis cliquez sur "OK".

De retour sur le wizard de configuration du Partner Link, "IWSAddition" est désormais disponible. Sélectionnez-le. La définition du type du PartnerLink apparaît dessous avec son opération et ses paramètres tels que nous les avons définis dans l'interface de notre service web. Cliquez sur "OK".

Un nouveau wizard vous demande de donner un nom au type de votre Partner Link. Nommez-le et cliquez sur "Next >".

Puis le wizard vous demande de sélectionner le "Port Type" de ce type de Partner Link et de lui donner un nom de rôle. Sélectionner IWSAddition et donner lui un nom de rôle ; "additionne"r par exemple. Il est possible de donner plusieurs rôle à votre PartnerLink type en cliquant sur "Next >". Pour notre exemple, chaque "Partner Link Type" n'a qu'un seul rôle, nous pouvons donc cliquer directement sur "Finish".

Votre Partner Link est désormais importé dans votre BPEL Process File. Il ne vous reste plus qu'à sélectionner son rôle, additionner dans le cas présent, dans l'onglet "Détails" des "Properties" du Partner Link.

Reproduisez exactement le même protocole d'import de Partner Link pour les services web multiplication et division.
