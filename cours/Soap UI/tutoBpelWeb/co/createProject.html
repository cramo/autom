Création du projet BPEL

Création du projet et configuration du runtime

Pour créer un projet BPEL, allez dans "File > New > Other..." et tapez "bpel" dans le champ de filtre. Sélectionnez "BPEL Project" puis cliquez sur "Next >"

Donnez un nom à votre projet puis cliquez sur "New Runtime...". Cela va ouvrir un wizard de configuration de Runtime pour ce projet. Nous allons pouvoir lier notre projet BPEL au serveur ODE que nous avons configuré au tout début de ce cours.

Dans la liste de serveurs proposés, cherchez le dossier "Apache" et sélectionnez "Apache ODE 1.x Runtime". Cliquez ensuite sur "Next >"

A côté du label "ODE's home directory :", cliquez sur "Browse..." et indiquez le chemin du dossier "ode" qui est apparu dans le répertoire "webapps" de votre serveur Tomcat tout à l'heure lorsque vous avez déployé le fichier "ode.war"

De même, à côté du label "Tomcat's home directory :", cliquez sur "Browse..." et indiquez le chemin d'accès du répertoire racine de votre serveur Tomcat.

N'oubliez pas de modifier aussi le port du serveur si vous n'utilisez pas le port par défaut (8080)

Cliquez ensuite sur "Finish"

De retour sur le wizard de création du projet, cliquez simplement sur "Finish"

Votre projet a été créé. Il ne contient pour le moment qu'un fichier bpelContent vide. Eclipse vous propose aussi de passer en perspective BPEL. Cliquez alors sur "Yes". Cette perspective est ergonomiquement conçu pour utiliser les vues "Design" de l'outil BPEL eclipse.

Création d'un BPEL Process File

Faites un clic droit sur le dossier bpelContent de votre projet Bpel puis sélectionnez "New > Other...", tapez "bpel" dans le champ de filtre. Sélectionnez "BPEL Process File" puis cliquez sur "Next >"

Donnez un nom et un namespace à votre process puis cliquez sur "Next >"

A côté du label 'Template :", sélectionnez "Synchronous BPEL Process". Modifiez le port de la "Service Address" si votre serveur ODE n'est pas sur le port 8080. Cliquez ensuite sur "Next >"

Le wizard vous demande ensuite l'emplacement dans lequel vous souhaitez placer votre BPEL Process File. Par défaut, eclipse le place dans le fichier" bpel content". Cela nous convient parfaitement. Cliquez sur "Finish".

Notre BPEL Process File est créé. Il s'agit du fichier dans le quel nous allons importer le partnerLink de notre orchestrateur, déclarer les variables dont nous avons besoins et programmer notre logique d'orchestration. Eclipse nous a généré un patron d'orchestration dans lequel est déjà déclaré un partnerLink qui correspond à l'opération de l'orchestrateur, ainsi que ses deux variables "input" et "output".
